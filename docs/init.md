### **核心纪律**：绝对不在特征计算中使用未来数据，坚持硬性过滤、概率排序和样本外验证，模型复杂度永远服务于样本外稳定性。

### 第1步：建立符合条件的初始股票池

（这一步是基础，直接卡掉不符合硬性条件的标的，后续所有操作都在这个池子里进行）

请用Python和akshare库，帮我构建一个A股可交易股票池的初始筛选模块，要求如下：

1. 获取沪深两市所有A股的实时行情和基本面数据（可使用akshare的stock_zh_a_spot_em()），提取以下字段：
  
  - 代码、名称
  - 总市值（单位：元）
  - 最新价
  - 成交量（手）和成交额（元，若只有成交量则用“成交量*最新价*100”估算）
  - 市盈率、市净率等可一并保留备用。
2. 执行严格的硬性过滤，只保留同时满足以下条件的股票，并保存为本地CSV文件（如"qualified_stock_pool.csv"）：
  
  - 总市值 >= 200亿
  - 当日成交额 > 5亿
  - 剔除股票代码以“688”开头的股票（科创板）
  - 剔除股票代码以“8”开头（且长度为6位）的股票（北交所）
  - 剔除ST、*ST股票（名称中包含“ST”）
  - 剔除已退市或长期停牌的股票（若状态字段不可用，先保留，后续通过日线数据完整性过滤）。
3. 代码要健壮，包含异常处理和请求延时，避免被封。并输出过滤前后的股票数量。
  

### 第2步：获取池内股票的深度历史日线数据

（只拉取通过硬性条件的股票，大幅节省时间和资源）

基于上一步生成的“qualified_stock_pool.csv”中的股票代码列表，执行：

1. 使用akshare的stock_zh_a_hist()函数，逐只获取日线数据（前复权），时间范围至少从2018-01-01到今天，确保模型有足够样本。
2. 数据列必须包含：日期、开盘、收盘、最高、最低、成交量、成交额。
3. 自动跳过获取失败或数据严重不足（如少于200个交易日）的股票，并记录日志。
4. 将每只股票的数据存为独立CSV，文件名用股票代码，目录为 "./stock_data/"。
5. 同时获取沪深300指数日线数据（代码000300，用stock_zh_index_daily），存入"./stock_data/HS300.csv"，作为市场基准。
6. 代码需有重试机制，间断运行，打印进度。

### 第3步：因子工程——构建专业化的特征向量

（这里要求向量化计算，禁止使用未来数据）

写一个因子计算模块，输入是单只股票的历史日线DataFrame（从./stock_data/加载），输出是一个包含所有因子的DataFrame，列名统一。要求：

1. 全部使用向量化pandas/numpy操作，禁止逐行循环。
2. 计算以下因子，且计算时严格使用到当日为止的数据（即，所有因子值在当日收盘后即可算得，不包含未来信息）：
  【趋势因子】
  - 收盘价站上5、10、20、60日均线的百分比位置（如：(close-ma5)/ma5）。
  - 均线排列情况：5>10>20记1，5<10<20记-1，其他0。
  - 近20日最高价创新高标记（当日最高>=前20日最高）。
  - MACD的DIF、DEA、MACD柱的值，以及DIF和DEA的正负。
    【量价因子】
  - 当日成交量与5日、10日、20日均量比。
  - 当日成交额与20日均成交额比。
  - 缩量程度（成交量比前一日缩量超过20%且收阳/收阴的组合标识）。
    【动量因子】
  - 5日、10日、20日累计涨跌幅。
  - 最近5日上涨天数占比。
  - RSI(14)。
    【波动及路径因子】
  - 真实波幅ATR(14)。
  - 当日最高-最低与开盘-收盘的比例关系（描述日内形态）。
    【市场环境因子】
  - 将沪深300指数的同期的20日涨幅作为一列添加到每日样本中（对应日期匹配）。
3. 所有因子必须处理空值和除零问题。返回的DataFrame包含原始日期、开高低收量和计算出的所有因子列。

### 第4步：定义无未来函数的上涨标签

（这是防止模型作弊的核心）

基于每只股票的日线数据，生成一个预测用的目标标签。要求：

1. 标签定义为：“未来2个交易日的累计涨幅超过3%”，记为1，否则为0。
  计算方法：用未来第2日的收盘价 / 当日收盘价 - 1 >= 0.03，则为1。
2. 标签列放在对应“当日”的样本行上。绝不能用当日或之前的数据计算，必须严格通过shift(-2)等方式创建。
3. 对于数据末尾无法产生标签的行（最后两天），直接丢弃（或标记NaN后删除）。
4. 返回增加了“target”列的DataFrame。

### 第5步：样本汇总与滚动训练框架

（使用LightGBM，采用严格的时间序列滚动）

将第3、4步生成的所有股票因子+标签数据合并成一张统一的大表。要求：

1. 加载所有处理好的股票文件，合并时每行加入股票代码列。
2. 按日期排序，确保时间先后正确。
3. 实现一个LightGBM滚动训练类，参数要求：
  - 训练窗口长度：600个交易日。
  - 重新训练频率：每120个交易日重新训练一次。
  - 每次训练后，对紧接着的未来20个交易日样本进行概率预测。
  - 训练时，去除“日期”和“股票代码”列作为特征，仅用因子列。标签为“target”。
  - 输出每期在预测区间上的AUC值，打印出来监控模型稳定性。
4. 使用早停和交叉验证（用sklearn的TimeSeriesSplit）防止过拟合。若早停选出的best_iteration过低，最终生产模型不得直接退化为极少树数，应设置最小最终树数下限（如80），低于下限时回退使用配置的完整树数重新训练。
5. 最后，使用最新的完整模型，对最新一个交易日所有股票样本进行预测，生成“上涨概率”列。输出概率前20的股票代码及其概率，保存为CSV。

### 第6步：植入硬性过滤+固定Top10信号生成器

（在你日常运行预测时，再次强制应用市值/成交额/板块条件，然后固定输出池内概率最高的10只股票）

在每日生成选股列表的阶段（接在第5步概率输出之后），创建一个最终的“信号过滤器”模块，流程如下：

1. 加载之前第1步保存的“qualified_stock_pool.csv”最新文件，获取符合条件的股票池（已满足市值≥200亿、成交额>5亿、非科创板/北交所/ST）。
2. 只允许在该池内的股票保留概率值，池外股票即使有概率也直接剔除。
3. 最终信号阶段不再根据沪深300的20日线、60日线调节持仓数量，也不再因为市场环境偏空而空仓。HS300.csv仍可保留用于市场因子计算和回测对照，但不得在最终信号阶段削减候选数量。
4. 默认不设置最低概率阈值；如需实验阈值，只能作为可选参数显式传入。日常运行按概率从高到低排序，固定取前10只股票，作为当日最终交易信号。
5. 输出最终名单为CSV和Excel，包含：股票代码、名称、概率、市值、成交额、生成时间戳和规则说明。

### 第7步：严谨的T+1回测模拟

（贴近真实交易，包含涨停买不进等情况）

基于第6步逐日生成的历史选股列表（你需要用滚动方式从过去某天到现在遍历生成每日组合），构建一个T+1回测引擎。要求：

1. 交易规则：
  - 买入价：次日开盘价。如果开盘即封死涨停（开盘价=涨停价且成交量极小），则判定买入失败，该次交易不成交，资金留作现金。
  - 卖出价：买入后第2个交易日收盘价卖出。
  - 单只股票仓位：等权分配，每日总资金平均分配给当日入选的10只股票；若因数据缺失等异常导致不足10只，则在实际入选股票之间等权分配。
  - 手续费：买入0.03%，卖出0.13%（含印花税），最低5元。
2. 计算并绘制：
  - 策略累计净值曲线 vs 沪深300累计净值曲线。
  - 年化收益率、最大回撤、夏普比率、胜率（盈利次数/总交易次数）、盈亏比（平均盈利/平均亏损）、交易次数。
3. 保证回测数据无未来信息：每日选股时只能使用当日及之前的数据，这个限制在生成每日持股列表时就要遵守。你可以写个函数，从数据起始日开始，每天喂入当天之前的所有数据做预测并记录选股，循环往复。
4. 代码要能完整运行并输出回测报告。
