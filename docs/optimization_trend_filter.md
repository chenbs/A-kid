# 选股优化记录：趋势硬过滤（剔除下跌趋势股票）

> 本文档记录 2026-06-28 针对"选出的票很多是明显下跌趋势"问题的排查与优化。
> 目标：找出【下两个交易日累计涨幅 >= 3% 概率最大】，且处于上升趋势中的股票。

## 一、问题现象

按 `docs/init.md` 跑通后，最终 Top10 信号中混入明显下跌趋势的股票，例如：

| 代码 | 名称 | 近20日涨幅 | close/ma20 | 入选概率 |
|------|------|-----------|-----------|---------|
| 300255 | 常山药业 | **-41.5%** | -30.5% | 0.7403 |
| 600988 | 赤峰黄金 | **-23.7%** | -15.1% | 0.7001 |

这与"只要上升趋势中的高概率上涨股"的真实意图相悖。

## 二、根因排查

排查结论：**模型本身没坏**（历史滚动预测中每日 Top10 实际命中率 33% vs 基准 18%，概率十分位与命中率单调正相关），真正的问题在标签定义与信号过滤：

### 根因 1：标签本质是"波动率信号"，不是"趋势信号"

`target = 未来2日累计涨幅 >= 3%`。按因子分桶统计实际命中率：

```
波动率(volatility_20d)分桶: 最低档 8.2%  →  最高档 32.3%   (单调上升)
ATR%(atr_14_pct)分桶:       最低档 6.8%  →  最高档 33.0%   (单调上升)
```

模型为了最大化"2日涨3%"的概率，核心学到的是"挑波动大的票"。

### 根因 2：暴跌票命中率反而是第二高（U 型）

```
20日动量(return_20d)分桶: 最跌档 25.6% | 中间档 ~15% | 最涨档 29.5%   (U型)
站上20日线幅度分桶:        最低档 27.3% | 中间档 ~16% | 最高档 29.3%   (U型)
```

原因：暴跌的票波动大、容易超跌反弹 3%，正好被模型当成高概率标的。

### 根因 3：信号阶段删光了所有趋势/市场过滤

`docs/init.md` 第6步要求"不再根据沪深300调节持仓、不再因偏空空仓"，导致下跌趋势股毫无阻拦地进入 Top10。

## 三、优化方案：趋势硬过滤

在**信号阶段（第6步）**加入硬过滤，只保留健康上升趋势的股票：

```
站上20日线 (close_ma20_pos > 0)
且 站上60日线 (close_ma60_pos > 0)
且 近20日累计涨幅为正 (return_20d > 0)
```

选择"信号阶段过滤"而非"改标签"的理由：
- 你的目标就是"2日涨≥3%概率最大"。高动量股命中率最高，限制涨幅上限反而把命中率从 0.329 拉低到 0.309。
- 真正的病是"下跌票被选入"，趋势硬过滤可直接根除，且更透明、可解释。
- 改标签（如改成相对沪深300超额收益）会同时削弱有效的动量信号，得不偿失。

## 四、回测验证

基于历史滚动预测（`model_outputs/rolling_predictions.csv`）对比加过滤前后的真实 2 日收益：

| 方案 | 每日Top10命中率 | 每期平均2日收益 | 胜率 | 累计收益(近似) |
|------|----------------|----------------|------|---------------|
| 无过滤 | 0.3293 | 0.930% | 55.7% | 310% |
| **趋势过滤** | **0.3425** | **0.975%** | 54.5% | **339%** |

过滤后每日仍有约 179~207 只候选，足以选出 Top10。

## 五、代码改动

### `rolling_lightgbm_train.py`
- 新增 `TREND_OUTPUT_COLUMNS = (close_ma20_pos, close_ma60_pos, ma_arrangement, return_20d, return_5d)`。
- `predict_latest()` 在 `latest_predictions_all.csv` 中一并输出这些趋势列，供第6步过滤使用。

### `signal_filter.py`
- `SignalConfig` 新增 `trend_filter: bool = True`（默认开启）。
- 新增 `apply_trend_filter()`：执行"站上20/60日线 + 20日涨幅>0"过滤。
- 命令行新增 `--no-trend-filter` 可关闭过滤。
- 若预测文件缺少趋势列，自动跳过过滤并告警（需用新版第5步重新生成预测）。

## 六、运行方式

```bash
# 第5步：滚动训练 + 生成带趋势列的最新预测
.venv/Scripts/python.exe rolling_lightgbm_train.py

# 第6步：应用趋势硬过滤，生成最终 Top10 信号（默认开启过滤）
.venv/Scripts/python.exe signal_filter.py

# 如需关闭趋势过滤（退回原始行为）
.venv/Scripts/python.exe signal_filter.py --no-trend-filter
```

## 七、已知权衡

趋势过滤后的 Top10 全部是已经大涨过的强势股（如近20日 +50%~90%），存在追高回调风险。但这正是"未来2日涨≥3%概率最大"的数学结果——动量越强，2日内继续涨3%的历史概率越高。

若真实意图偏向"还没涨、即将启动"的票，那是另一个目标（需更换标签与因子体系），与"概率最大"在数学上是矛盾的，需另行设计。
